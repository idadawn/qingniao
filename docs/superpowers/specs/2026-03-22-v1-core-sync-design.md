# V1 核心同步功能设计

## 概述

为「青鸟」浏览器扩展设计 V1 版本的核心功能：实现浏览器书签与飞书多维表格之间的双向同步。

## 背景

用户希望将浏览器书签同步到飞书多维表格中，以便进行结构化管理、检索和协作。同时，用户也希望在飞书多维表格中对书签进行整理后，能够同步回浏览器。

## 核心功能

1. **飞书授权与配置**：用户在扩展的 Options 页面配置飞书应用的 App ID、App Secret 以及目标多维表格的 URL。
2. **全量初始化导入**：首次使用时，将浏览器中的所有书签全量导入到飞书多维表格中。
3. **增量双向同步**：
   - **浏览器 → 飞书**：监听浏览器书签的增删改事件，实时推送到飞书多维表格。
   - **飞书 → 浏览器**：用户在 Popup 界面点击"立即同步"按钮，扩展拉取飞书多维表格的最新数据，与本地书签进行比对并合并。

## 数据结构设计

### 飞书多维表格结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | 文本 | 书签名称 |
| URL | 超链接 | 书签地址 |
| 文件夹路径 | 文本 | 如 `/工作/前端` |
| 创建时间 | 日期 | 书签添加时间 |
| 最后修改时间 | 日期 | 用于冲突判断 |
| 来源浏览器 | 单选 | Chrome / Edge / Safari |
| 浏览器内部 ID | 文本 | 用于双向映射，隐藏列 |
| 同步状态 | 单选 | 正常 / 待同步 / 冲突 |

## 交互流程

### Popup 界面

1. **未连接状态**：提示用户前往 Options 页面配置飞书账号。
2. **已连接状态**：显示本地书签和飞书表格的书签数量，提供"立即同步"按钮，显示上次同步时间。
3. **同步中状态**：显示同步进度和动画。
4. **同步完成状态**：显示同步结果摘要（新增、更新、删除的数量）。

### Options 页面

1. **飞书配置**：输入 App ID、App Secret、多维表格 URL。
2. **同步设置**：选择冲突处理策略（默认以时间戳较新者为准）。
3. **危险操作**：提供清空飞书表格并重新全量同步的选项。

## 冲突处理策略

当同一书签在浏览器和飞书多维表格中同时被修改时，默认以**最后修改时间较新**的一方为准。
