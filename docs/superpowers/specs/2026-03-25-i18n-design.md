# 设计文档：国际化（i18n）支持

**日期**：2026-03-25
**状态**：草稿

## 背景

当前 `manifest.json` 中 `name` 和 `description` 字段为硬编码中文字符串，导致：

1. Edge Add-ons Partner Center 仅能识别到一个语言（单 locale），无法添加多语言 Store Listing
2. Chrome Web Store 只显示 en 商店页，无法单独配置中文描述
3. 扩展 UI（Popup/Options 页面文字）未走国际化，未来切语言需改 HTML

## 目标

- Edge Add-ons 能识别到 `zh_CN` 和 `en` 两个 locale，分别填写对应商店详情
- `manifest.json` 的 `name` / `description` 通过 `__MSG_xxx__` 占位符动态加载
- Popup / Options 页面 UI 文字通过 `chrome.i18n.getMessage()` 加载，支持中英切换
- 默认语言为 `zh_CN`

## 数据结构

```
src/
  _locales/
    zh_CN/
      messages.json   ← 中文（默认）
    en/
      messages.json   ← 英文
```

### messages.json 结构

```json
{
  "extensionName": {
    "message": "青鸟 - 书签同步助手"
  },
  "extensionDescription": {
    "message": "将浏览器书签与飞书多维表格双向同步，让你的知识永远触手可及。"
  }
}
```

## manifest.json 变更

| 字段 | 变更前 | 变更后 |
|------|--------|--------|
| `name` | `"青鸟 - 书签同步助手"` | `"__MSG_extensionName__"` |
| `description` | `"将浏览器书签..."` | `"__MSG_extensionDescription__"` |
| `default_locale` | 无 | `"zh_CN"` |

## 范围说明

**本期（MVP）**：仅 manifest 层 i18n，满足商店多语言 listing 要求。
**下期（可选）**：Popup / Options UI 文字也走 `chrome.i18n.getMessage()`。

## 风险

- `default_locale` 设置为 `zh_CN` 后，`_locales/zh_CN/messages.json` 必须存在，否则扩展加载失败
- `en` locale 的 messages.json 必须包含 `name` / `description` 对应的 key，否则 Edge 商店英文 listing 名称显示异常
