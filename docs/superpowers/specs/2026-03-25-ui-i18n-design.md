## 1. 概述

### 1.1 目标
为「青鸟」浏览器扩展实现完整的 UI 国际化支持，使界面文本能够根据浏览器语言设置自动切换中英文。

### 1.2 范围
- ✅ Popup 弹窗页面
- ✅ Options 设置页面
- ✅ 后台通知（notifications）
- ✅ 工具提示（tooltips）
- ✅ 动态生成的文本

### 1.3 语言支持
| 语言代码 | 语言 | 优先级 |
|---------|------|--------|
| zh_CN | 简体中文 | P0（默认） |
| en | 英语 | P0 |

## 2. 技术架构

### 2.1 文件结构

```
src/
├── _locales/
│   ├── zh_CN/
│   │   └── messages.json    # 中文语言包
│   └── en/
│       └── messages.json    # 英文语言包
├── utils/
│   └── i18n.js              # i18n 工具函数
├── popup/
│   ├── popup.html           # 添加 data-i18n 属性
│   └── popup.js             # 引入 i18n 模块
├── options/
│   ├── options.html         # 添加 data-i18n 属性
│   └── options.js           # 引入 i18n 模块
└── manifest.json            # 已配置 default_locale
```

### 2.2 messages.json 格式规范

```json
{
  "keyName": {
    "message": "实际显示的文本",
    "description": "给翻译人员的说明（可选）"
  },
  "keyWithPlaceholder": {
    "message": "新增 $count$ 条书签",
    "description": "$count$ 会被替换为数字",
    "placeholders": {
      "count": {
        "content": "$1",
        "example": "5"
      }
    }
  }
}
```

**命名规范**：
- 使用 camelCase
- 按功能模块分组，用前缀区分：
  - `popup*` - Popup 页面
  - `options*` - Options 页面
  - `common*` - 通用文本
  - `notify*` - 通知消息

## 3. 实现方案

### 3.1 核心工具函数

**文件**：`src/utils/i18n.js`

```javascript
/**
 * 获取本地化消息
 * @param {string} key - 消息 key
 * @param {string[]} substitutions - 替换参数
 * @returns {string}
 */
export function t(key, ...substitutions) {
  return chrome.i18n.getMessage(key, substitutions) || key;
}

/**
 * 初始化页面 i18n
 * 自动替换所有带有 data-i18n 属性的元素
 */
export function initI18n() {
  // 替换文本内容
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.getAttribute('data-i18n');
    const message = t(key);
    if (message) {
      el.textContent = message;
    }
  });

  // 替换 placeholder
  document.querySelectorAll('[data-i18n-placeholder]').forEach(el => {
    const key = el.getAttribute('data-i18n-placeholder');
    const message = t(key);
    if (message) {
      el.placeholder = message;
    }
  });

  // 替换 title 属性
  document.querySelectorAll('[data-i18n-title]').forEach(el => {
    const key = el.getAttribute('data-i18n-title');
    const message = t(key);
    if (message) {
      el.title = message;
    }
  });
}
```

### 3.2 HTML 集成方式

#### 基础用法

```html
<!-- 静态文本 -->
<span data-i18n="popupNotConnected">尚未连接飞书</span>

<!-- 动态内容（JS 中更新） -->
<span id="status-text">尚未连接飞书</span>

<!-- Placeholder -->
<input data-i18n-placeholder="optionsAppIdPlaceholder" placeholder="cli_xxxxxxxxxxxxxxxx" />

<!-- Title 属性 -->
<button data-i18n-title="popupSettingsTitle" title="设置">
  <svg>...</svg>
</button>
```

#### 初始化

```javascript
// popup.js / options.js
import { initI18n, t } from '../utils/i18n.js';

document.addEventListener('DOMContentLoaded', () => {
  // 初始化静态文本
  initI18n();
  
  // 动态文本使用 t() 函数
  updateDynamicContent();
});

function updateDynamicContent() {
  const count = 5;
  document.getElementById('added-count').textContent = 
    t('popupAddedX', count.toString());
}
```

### 3.3 动态文本处理

**原始代码**：
```javascript
element.textContent = `新增 ${count} 条`;
```

**国际化后**：
```javascript
// messages.json
// "popupAddedX": { "message": "新增 $1$ 条" }

element.textContent = t('popupAddedX', count.toString());
```

**注意事项**：
- Chrome i18n 使用 `$1$`, `$2$` 作为占位符
- 参数必须是字符串类型，数字需要 `toString()`

## 5. 开发规范

### 5.1 添加新文本的流程

1. **在中文 messages.json 中添加 key 和文本**
2. **在英文 messages.json 中添加翻译**
3. **在 HTML 中添加 `data-i18n` 属性**
4. **在 JS 中使用 `t()` 函数**
5. **测试中英文显示**

### 5.2 占位符使用规范

```json
// ✅ 正确
{
  "message": "新增 $1$ 条书签",
  "placeholders": {
    "count": {
      "content": "$1"
    }
  }
}

// ❌ 错误 - 不要使用 named placeholder 在 message 中
{
  "message": "新增 $count$ 条书签"
}
```

### 5.3 HTML 转义

如果文本包含 HTML，使用 JS 处理：

```javascript
// 不要这样
// <span data-i18n="htmlContent">

// 应该这样
const html = `<strong>${t('important')}</strong>: ${t('message')}`;
element.innerHTML = html;
```
